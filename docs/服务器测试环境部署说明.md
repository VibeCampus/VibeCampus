# VibeCampus 服务器测试环境部署说明

面向 Ubuntu 22.04 云服务器，本文只保留已验证成功的路线：

- 本地构建制品 + 本地构建镜像 + 打包
- 服务器导入镜像并启动（服务器不构建）

---

## 0. 先看整体架构（为什么这么做）

这套方案把“重活”放在本地（编译、打镜像），把服务器变成“运行节点”（只加载镜像并运行容器）。

好处：

- 服务器部署步骤少，出错点少
- 服务器不需要 Maven / Node 环境
- 版本可控：本地产出就是线上运行版本

---

## 1. Dockerfile 与 Compose 全说明

## 1.1 本次真正使用的文件（`deploy/test/prebuilt`）

- `docker-compose.images.yml`
  - **做什么**：定义线上运行的 3 个服务 `mysql`、`backend`、`web`
  - **为什么需要**：统一声明容器关系、环境变量、端口、卷挂载
  - **关键点**：
    - 用 `image:` 而不是 `build:`，表示服务器直接用已导入镜像
    - `backend` 和 `web` 镜像标签由 `IMAGE_TAG` 控制
    - `MYSQL_INIT_SQL` 默认 `./schema.mysql.sql`，首次建卷自动初始化

- `backend/Dockerfile`
  - **做什么**：把 `app.jar` 打包成后端运行镜像
  - **为什么需要**：把 Java 运行环境和 JAR 固化，服务器无需 Java/Maven
  - **关键点**：
    - `FROM eclipse-temurin:21-jre-jammy`：只保留 JRE 运行时
    - `COPY app.jar /app/app.jar`：把本地编译好的应用放入镜像
    - `ENTRYPOINT ["java","-jar","/app/app.jar"]`：容器启动即运行后端

- `web/Dockerfile`
  - **做什么**：把前端 `dist` 和 Nginx 配置打包成前端镜像
  - **为什么需要**：前端静态资源与反代配置随镜像发布，避免服务器手工配 Nginx
  - **关键点**：
    - `FROM nginx:1.27-alpine`：体积小，启动快
    - `COPY default.conf`：配置 `/api` 反代到 `backend:8080`
    - `COPY dist/`：部署构建后的静态页面

- `package.ps1` / `package.sh`
  - **做什么**：自动完成本地编译、构建镜像、导出 tar、打部署包
  - **为什么需要**：减少手工命令，保证每次发布流程一致

## 2. 快速部署流程（复制即用）

### 2.1 本地（Windows）

```powershell
# 在仓库根目录执行
.\deploy\test\prebuilt\package.ps1 -Tag test

# 上传部署包到服务器（示例）
scp .\deploy\test\prebuilt\_bundle\prebuilt-test.tgz root@<服务器公网IP>:~/
```

这一步的含义：

- `-Tag test` 会产出 `vibecampus-backend:test` 与 `vibecampus-web:test`
- 生成 `prebuilt-test.tgz`，里面包含 compose、env 模板、schema、镜像 tar

### 2.2 服务器（Ubuntu）

```bash
# 首次机器先安装 Docker
sudo apt update
sudo apt install -y docker.io docker-compose-v2
sudo systemctl enable --now docker

# 解压并部署
cd ~
tar -xzf prebuilt-test.tgz
cd vibecampus-prebuilt
cp .env.example .env
vim .env
docker load -i _bundle/images-test.tar
docker compose -f docker-compose.images.yml --env-file .env up -d
```

这一步的含义：

- `docker load`：把本地打好的镜像导入服务器
- `docker compose up -d`：按 compose 编排启动全部服务

## 3. 每一步详细说明（做什么 + 为什么 + 成功标准）

### 步骤 1：准备服务器网络

做什么：

- 安全组放行 22（SSH）和 80（HTTP）

为什么：

- 22 用于远程管理
- 80 用于浏览器访问前端页面

成功标准：

- 能 SSH 登录
- 浏览器能访问服务器 80 端口

### 步骤 2：安装 Docker

做什么：

```bash
sudo apt update
sudo apt install -y docker.io docker-compose-v2
sudo systemctl enable --now docker
docker --version
docker compose version
```

为什么：

- `docker.io` + `docker-compose-v2` 是 Ubuntu 22.04 上最稳定的直接安装方式

成功标准：

- `docker --version` 和 `docker compose version` 有正常输出

### 步骤 3：本地打包

做什么：

```powershell
.\deploy\test\prebuilt\package.ps1 -Tag test
```

为什么：

- 一次性完成：
  - Maven 打后端 JAR
  - npm 打前端 `dist`
  - 构建后端/前端镜像
  - 拉取并打包 MySQL 镜像
  - 输出可部署压缩包

成功标准：

- 输出目录出现 `deploy/test/prebuilt/_bundle/prebuilt-test.tgz`

### 步骤 4：上传部署包

做什么：

```bash
scp deploy/test/prebuilt/_bundle/prebuilt-test.tgz root@<服务器IP>:~/
```

为什么：

- 只传一个文件，避免漏传 `app.jar`、`dist`、compose 等

成功标准：

- 服务器 `~/prebuilt-test.tgz` 存在

### 步骤 5：服务器解压 + 配置环境变量

做什么：

```bash
tar -xzf prebuilt-test.tgz
cd vibecampus-prebuilt
cp .env.example .env
vim .env
```

为什么：

- `.env` 是部署时真正生效的配置（账号密码、端口、镜像标签）

`.env` 必改项：

```dotenv
MYSQL_ROOT_PASSWORD=强密码
MYSQL_PASSWORD=强密码
WEB_PORT=80
IMAGE_TAG=test
MYSQL_INIT_SQL=./schema.mysql.sql
```

成功标准：

- `vibecampus-prebuilt/.env` 文件存在并已保存

### 步骤 6：导入镜像 + 启动服务

做什么：

```bash
docker load -i _bundle/images-test.tar
docker compose -f docker-compose.images.yml --env-file .env up -d
```

为什么：

- `docker load` 把离线镜像加载到服务器
- `compose up` 按依赖顺序启动 `mysql`、`backend`、`web`

成功标准：

- `docker load` 输出 `Loaded image: ...`
- `compose up` 无错误返回

## 4. 如何判定“部署成功”

```bash
docker compose -f docker-compose.images.yml --env-file .env ps
docker compose -f docker-compose.images.yml --env-file .env logs --tail=200 backend
docker compose -f docker-compose.images.yml --env-file .env logs --tail=200 web
curl -I http://127.0.0.1/
curl -I http://127.0.0.1/api
```

判定标准：

- `mysql` 为 `healthy`
- `backend`、`web` 为 `Up`
- `curl` 不出现 502/504/connection refused
- 浏览器可打开 `http://<公网IP>`（或 `http://<公网IP>:<WEB_PORT>`）

## 5. 更新版本（重复部署）

做什么：

1. 本地重新执行 `package.ps1 -Tag test`
2. 重新上传新的 `prebuilt-test.tgz`
3. 服务器执行：

```bash
tar -xzf prebuilt-test.tgz
cd vibecampus-prebuilt
docker load -i _bundle/images-test.tar
docker compose -f docker-compose.images.yml --env-file .env up -d
```

为什么：

- 保持同一 tag（`test`）时，导入镜像后 compose 会按新镜像重建容器

## 6. 常见问题与解释

- 本地 `docker pull` 超时
  - 含义：本机无法访问 Docker Hub
  - 处理：配置 Docker Desktop 镜像加速或 HTTPS 代理

- 服务器提示缺少 `.env`
  - 含义：compose 读取环境文件失败
  - 处理：先 `cp .env.example .env` 再启动

- 页面打开但接口失败
  - 含义：前端容器正常，后端或数据库连接异常
  - 处理：看 `backend` 日志与 `.env` 数据库配置

- 修改 `schema.mysql.sql` 不生效
  - 含义：MySQL 初始化脚本只在首次建卷执行
  - 处理：重建卷（会清空数据）：
    ```bash
    docker compose -f docker-compose.images.yml --env-file .env down -v
    docker compose -f docker-compose.images.yml --env-file .env up -d
    ```
