# VibeCampus 数据库说明文档（MySQL 8+）

本设计依据 `VibeCampus-Frontend` 当前的接口与页面字段使用方式推导，目标是**覆盖前端现有功能**并为后端后续落地留足扩展空间。

## 1. 约定与通用规范

- **字符集**：`utf8mb4`（支持表情与多语言）。
- **存储引擎**：`InnoDB`（事务、行锁、外键）。
- **命名风格**：表名/字段名 `snake_case`；后端可使用 MyBatis `map-underscore-to-camel-case` 映射为 `camelCase`。
- **主键**：统一 `BIGINT UNSIGNED` 自增 `id`。
- **时间字段**：
  - `created_at`：创建时间
  - `updated_at`：更新时间（自动 on update）
  - `deleted_at`：软删除（需要保留历史/审计的表使用；不需要的表可不加）

对应可执行 DDL 见：`schema.mysql.sql`。

## 2. 业务域与表清单

- **用户与权限**：`users`、`roles`、`permissions`、`user_roles`、`role_permissions`
- **分类**：`categories`
- **内容**：`posts`、`post_media`、`comments`
- **互动**：`post_likes`、`post_favorites`、`comment_likes`、`follows`
- **消息中心**：`notifications`、`system_messages`、`chat_sessions`、`chat_messages`
- **运营与风控**：`announcements`、`sensitive_words`
- **审计日志**：`operation_logs`、`login_logs`

## 4. 前端字段映射要点（高一致性）

前端使用到的关键字段（简化）：

- **用户**：`id`、`username`、`avatar`（建议后端返回为 `avatar`，数据库用 `avatar_url`）
- **帖子**：`id`、`author`（对象）/`authorId`、`category`、`content`、`anonymous`、`images[]`、`likes`、`comments`、`time`
- **评论**：`id`、`postId`、`parentId`、`author`、`content`、`likes`、`time`、`replies[]`
- **通知**：`type(like/comment/reply)`、`userId`、`content`、`time`、`read`
- **私信**：会话列表（最后一条消息、未读数）、消息列表（from/content/time）
- **热榜**：`id + title`

因此数据库层建议：

- **计数字段做冗余**：`posts.like_count/comment_count/favorite_count`、`comments.like_count`（避免频繁聚合）。
- **媒体独立表**：`post_media` 支持 0..N 图片/视频；接口层可组装为 `images[]` + `video`（若需要）。
- **评论树**：用 `comments.parent_id` 表示回复关系；接口层按 `post_id` 拉取后组装树或分页策略。
- **热榜**：`posts.hot_score` 做可快速排序字段（同步或异步维护）。
